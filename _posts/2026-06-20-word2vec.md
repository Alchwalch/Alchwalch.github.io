## NNLM

Word2vec (CBOW,skip-gram)모델은 **NNLM의 문제점을 해결**하기 위해 만들어 졌다. $E$를 에포크라하고 $T$를 학습시킬 단어 개수라 하며 $Q$를 아키텍처에서 걸리는 시간복잡도라고 했을 때, 다음과 같이 표현할 수 있다.

$$O(E \times T \times Q)$$

임베딩을 학습시키는 모델들은 우리가 구할려고 하는 단어(target)에서 주변에 있는 단어들(context)로 학습을 시킨다. NNLM은 맥락단어의 각각의 분산표현을 이어붙여(concatenate)서 *projection layer*를 만들고 projection layer을 만들고 projection layer에서 hidden layer로 fc를 돌리고 마지막으로 hidden layer에서 전체 단어 종류의 수에 맞게 또 fc를 돌려서 단어 각각의 확률을 softmax와 cross-entropy로 구한다. 

![word2net1](assets/img/word2vec1.png)

마지막에 타겟 단어와 비교해서 학습을 시키는 방식이다. $N$을 맥락단어의 개수라 하고 $D$를 단어 벡터의 차원의 수, $H$를 은닉층의 차원, $V$를 모든 단어 종류 개수라고 했을 때 다음과 같이 표현할 수 있다.

$$Q = N \times D + N \times D \times H + H \times V$$

하지만 여전히 $H$때문에 복잡도가 늘어나므로 논문에서는 **아예 hidden-layer를 없애자**고 말을 한다. 그러면 $Q$를 아래와 같이 다이어트 시킬 수 있기 때문이다.

$$Q = N \times D + D \times V$$

인풋 데이터의 임베딩을 concatenate시킨 것이 아니라 하나로 합쳐서 평균값을 가진 하나의 은닉층을 만들고 다시 fc를 돌려서 단어의 확률을 구하는 방식으로 간다. 이렇게 context들을 통해 target을 보는 방식을 **CBOW**모델 이라한다. 반대로 target 단어의 임베딩을 통해 context들의 확률을 합산해서 구하는 방식은 **Skip-gram** 모델이라 한다. 아래 그림은 CBOW모델과 Skip-gram모델을 시각화한 것이다.

## Negative Sampling

그러나 100만개 정도나 되는 단어의 종류를 다 일일이 softmax를 하면 매우 시간이 오래 걸릴 것이다. 몬테카를로 처럼 타겟하고 몇몇개만 샘플링을 해서 softmax를 취하는 방식을 사용한다. 한 단어의 빈도를 $P(w)$라고 하고 $\theta$를 정답값으로 하고 그 외에 임의의 분포에서 추출한 오답분포를 n 이라 하면 다음과 같다. 

$$\frac{P_{\theta}(w)}{P_{\theta}(w) + k \cdot P_n(w)}$$

위 방식을 *Noise Constrastive Estimation*(NCE)라 한다. 그러나 우리는 단어의 임베딩의 품질을 높이면 장땡인데 굳이 분모계산을 일일이 다 하기에는 시간이 너무 걸린다. 그래서 아래와 같은 **Negative Sampling (NEG)**를 제안한다. $\sigma$는 sigmoid 함수이다.

$$\log \sigma\left({v'_{w_O}}^{\top} v_{w_I}\right) + \sum_{i=1}^{k} \mathbb{E}_{w_i \sim P_n(w)} \left[\log \sigma\left(-{v'_{w_i}}^{\top} v_{w_I}\right)\right]$$

정답에 관한 레이블 뿐만 아니라 분포를 통한 오답 레이블도 P에 따른 분포에 샘플링을 해서 부정값에 대해 목적함수 값에 기여한다.

확률 분포 P는 단순히 단어가 등장한 빈도를 의미한다고 보면된다. the가 corpus에서 1만개 단어중에 100개가 나왔다면 the는 0.01이다. 그러나 빈도가 너무 적은 단어들은 아예 샘플림이 안 될 수도 있다. 그래서 각각의 단어빈도에 0.75를 제곱하여 새로운 단어 빈도 만드는 것을 제안한다.

$$P'(w_i)=\frac{P(w_i)^{0.75}}{\sum^n_j(P(w_j)^{0.75})}$$

## Subsampling

빈도가 너무 높은 단어 (the나 a) 같은 경우는 데이터를 training하는데 시간을 많이 잡아먹을 수도 있고 의미론 적으로 크게 중요하지 않은 데이터일 가능성이 높다. 그래서 t=1e-5로 threshold로 잡고 t 보다 빈도가 높은 단어들을 아래와 같은 식으로 삭제(subsampling)될 확률을 정의한다. 

$$f(w_i) = 1 - \sqrt{\frac{t}{P(w_i)}}$$

이러면 속도도 빨라지고 성능이 좋아진다고 한다.

## 구현

구현은 먼저 numpy 부터 구현을 할 것이다. 먼저 Embedding 층에 대해 구현을 한 다음 Negative sampling, 그리고 CBOW, skip-gram모델을 각각 구현할 것이다.

### Embedding

Embedding 층을 구현하기 전에 탄생 배경부터 간단히 훑어보자. 원래는 단어마다 라벨을 지정해서 원핫 인코딩으로 변환하고 가중치와 내적하는 방식으로 계산을 했다. 그러나 단어 개수가 많아짐에 따라 가중치를 각각 계산하는것에 무리가 있었다. 그래서 원 핫 인코딩을 빼버리고 임베딩 벡터가 담겨있는 가중치에서 시작하자는 것이 배경이다. 그러므로 Embedding은 원래 가중치였으므로 params,grads가 있어야 되고 params에서 단어 라벨에 해당하는 정보를 꺼내야 되기 때문에 idx도 초기화 한다. 코드로 구현하면 아래와 같다.

```python
class Embedding:
  def __init__(self,W):
    self.params=[W]
    self.grads=[np.zeros_like(W)]
    self.idx=None

  def forward(self,idx):
    W, =self.params
    self.idx=idx
    out=W[idx]
    return out

  def backward(self, dout):
    dW, = self.grads
    dW[...] = 0
    np.add.at(dW, self.idx, dout)
    return None
```

forward에서 W,로 self.params=[W]의 W를 꺼내고 idx에 해당하는 벡터를 꺼내는 것을 볼 수 있다. backward에서 dW가 self.grads와 같은 주소를 공유하는 것을 볼 수 있고 dW[...]=0 를 함으로써 사전에 학습되었던 grads값을 0으로 초기화 시키는 것을 볼 수 있다. pytorch에서 optimizer.zero_grad()와 같은 역할이라 볼 수 있다.

### Negative Sampling

$$\log \sigma\left({v'_{w_O}}^{\top} v_{w_I}\right) + \sum_{i=1}^{k} \mathbb{E}_{w_i \sim P_n(w)} \left[\log \sigma\left(-{v'_{w_i}}^{\top} v_{w_I}\right)\right]$$

Negative Sampling을 전체적으로 보기전에 파트별로 끊어서 보는것이 좋다. 먼저 $\log \sigma$ (Sigmoid 함수와 cross-entropy)부분을 먼저보고 $w_i \sim P_n(w)$인 샘플링을 어떻게 처리하는지 구현해보자.

![word2net3](assets/img/word2vec3.png)

위 그림은 CBOW모델을 기반으로 그린 모델이다. 위 그림에선 Negative sampling 부분은 안 넣었고 정답 라벨에 해당할 때만 간단히 시각화 해서 보여주었다. (negative sampling을 표현한 그림은 아래에 나온다.) $W_{in}$ 부분은 분산표현 부분이고 그것과 다른 가중치인 $W_{out}$에서 단어에 해당하는 임베딩 벡터와 은닉층의 벡터와 내적을 한다. 그러면 단어 데이터를 하나씩 처리했을 때, 1x1 차원의 데이터가 나오게 된다. 즉 숫자 하나로 만들어서 sigmoid 함수에 넣을 수 있게 만든것이다. 단어 전체가 아니라 단어 하나하나 마다 비교하는 것이다. 

$W_{out}$도 임베딩된 레이어이기 때문에 해당하는 인덱스를 꺼내와서 내적을 해야된다. 이를 계산하기 위한 class는 다음과 같다.

```python
class EmbeddingDot:
    def __init__(self, W):
        self.embed = Embedding(W)
        self.params = self.embed.params
        self.grads = self.embed.grads
        self.cache = None

    def forward(self, h, idx):
        target_W = self.embed.forward(idx)
        out = np.sum(target_W * h, axis=1)

        self.cache = (h, target_W)
        return out

    def backward(self, dout):
        h, target_W = self.cache
        dout = dout.reshape(dout.shape[0], 1)

        dtarget_W = dout * h
        self.embed.backward(dtarget_W)
        dh = dout * target_W
        return dh
```

forward에서 h(은닉층) 벡터와 target_W 벡터를 내적하는 것을 볼 수 있고 cache로 따로 저장해서 backward에서 써먹는것을 볼 수 있다. self.embed.backward(dtarget_W)에서 idx는 따로 안 넣어도 이미 forward에서 idx가 기억되어있다. 

이제 정답 레이블외에 negative sampling을 샘플된 오답레이블들이 있을 때 어떻게 처리하는지 보자. 

![word2net4](assets/img/word2vec4.png)

위와 같이 정답레이블이 아닌 경우에는 정답 라벨을 0으로 sigmoid with loss 레이어에 입력을 한다. 그렇지 않은 경우에는 1. 즉, 위 식을 빌려서 정답일때는 $\log \sigma\left({v'_{w_O}}^{\top} v_{w_I}\right)$를 오답일 때는 $\log \sigma\left(-{v'_{w_i}}^{\top} v_{w_I}\right)\right$를 하는것이다.

시그모이드 부분 부터 처리하면 아래와 같은 객체로 표현할 수 있다.

```python
class SigmoidWithLoss:
    def __init__(self):
        self.params, self.grads = [], []
        self.loss = None
        self.y = None
        self.t = None

    def forward(self, x, t):
        self.t = t
        self.y = 1 / (1 + np.exp(-x))

        self.loss = cross_entropy_error(np.c_[1 - self.y, self.y], self.t)

        return self.loss

    def backward(self, dout=1):
        batch_size = self.t.shape[0]

        dx = (self.y - self.t) * dout / batch_size
        return dx
```

loss 값은 받아와야 되고 역전파 할때 y값과 t(라벨)값을 받아와야 한다. $(y-t) \times dout$. 그리고 forward 부분을 집중하면 label이 0일때 (오답) $1-y$ 값을 가리키고 반대로 label이 1 일때는 $y$를 가리키도록 배열을 설정한다. 외부의 일은 cross_entropy_error함수에서 처리하면 된다.

```python
def cross_entropy_error(y, t):
    if y.ndim == 1:
        t = t.reshape(1, t.size)
        y = y.reshape(1, y.size)

    # 정답 데이터가 원핫 벡터일 경우 정답 레이블 인덱스로 변환
    if t.size == y.size:
        t = t.argmax(axis=1)

    batch_size = y.shape[0]

    return -np.sum(np.log(y[np.arange(batch_size), t] + 1e-7)) / batch_size
```

return 부분을 잘 보면 미니배치에서 데이터 하나하나를 처리하기 위해 np.arrange()를 했고 t가 0이면 1-y로 t가 1이면 y를 가리키게 했다. 손실함수이므로 앞에 -를 붙이면 끝이난다.

이제 샘플링 부분을 어떻게 처리할지 생각해 보자. $w_i \sim P_n(w)$

$$P'(w_i)=\frac{P(w_i)^{0.75}}{\sum^n_j(P(w_j)^{0.75})}$$

먼저 위에 있는 식을 처리를 해야 된다. P는 전체 corpus에서 단어출현 수 (frequency)이다. __init__ 부분만 살펴보면 다음과 같다.

```python
class UnigramSampler:
    def __init__(self, corpus, power, sample_size):
        self.sample_size = sample_size
        self.vocab_size = None
        self.word_p = None

        counts = collections.Counter()
        for word_id in corpus:
            counts[word_id] += 1

        vocab_size = len(counts)
        self.vocab_size = vocab_size

        self.word_p = np.zeros(vocab_size)
        for i in range(vocab_size):
            self.word_p[i] = counts[i]

        self.word_p = np.power(self.word_p, power)
        self.word_p /= np.sum(self.word_p)
```

corpus에서 word_id마다 단어 개수를 저장하고 vocab_size에 word의 종류의 개수를 저장한다. 그리고 word_p에다가 counts를 옮기고 0.75제곱을 하고 확률로 바꾸는 것을 볼 수 있다. 이제 단어 각각 마다의 확률 분포를 표현할 수 있으니 그 분포에 따른 단어를 뽑는 함수를 만들면 된다. skip-gram이든 CBOW든 결과 단어들에 대한 target(정답)라벨들이 있을 텐데 target라벨에 해당하는 단어를 샘플링하면 안 되므로 그 확률은 0으로 처리하고 나머지를 조정하는 방식을 취한다. 그리고 조정된 확률 분포에 따라 무작위로 단어를 sample_size 만큼 뽑는다. 아래와 같이 코드를 표현할 수 있다. 

```python
class UnigramSampler:
    def __init__(self, corpus, power, sample_size):
        # (생략) ...

    def get_negative_sample(self, target):
        batch_size = target.shape[0]
        
        negative_sample = np.zeros((batch_size, self.sample_size), dtype=np.int32)

        for i in range(batch_size):
            p = self.word_p.copy()
            target_idx = target[i]
            p[target_idx] = 0
            p /= p.sum()
            negative_sample[i, :] = np.random.choice(self.vocab_size, size=self.sample_size, replace=False, p=p)
        
        return negative_sample
```

이렇게 하면 negative_sample에 각각에 대해서 오답 데이터에 대한 샘플링을 했고 그 데이터들로 sigmoidwithloss 계산을 진행하면 된다. 이걸 종합한 코드가 아래 class이다.

```python
class NegativeSamplingLoss:
    def __init__(self, W, corpus, power=0.75, sample_size=5):
        self.sample_size = sample_size
        self.sampler = UnigramSampler(corpus, power, sample_size)
        self.loss_layers = [SigmoidWithLoss() for _ in range(sample_size + 1)]
        self.embed_dot_layers = [EmbeddingDot(W) for _ in range(sample_size + 1)]

        self.params, self.grads = [], []
        for layer in self.embed_dot_layers:
            self.params += layer.params
            self.grads += layer.grads

    def forward(self, h, target):
        batch_size = target.shape[0]
        negative_sample = self.sampler.get_negative_sample(target)

        # 긍정적 예 순전파
        score = self.embed_dot_layers[0].forward(h, target)
        correct_label = np.ones(batch_size, dtype=np.int32)
        loss = self.loss_layers[0].forward(score, correct_label)

        # 부정적 예 순전파
        negative_label = np.zeros(batch_size, dtype=np.int32)
        for i in range(self.sample_size):
            negative_target = negative_sample[:, i]
            score = self.embed_dot_layers[1 + i].forward(h, negative_target)
            loss += self.loss_layers[1 + i].forward(score, negative_label)

        return loss

    def backward(self, dout=1):
        dh = 0
        for l0, l1 in zip(self.loss_layers, self.embed_dot_layers):
            dscore = l0.backward(dout)
            dh += l1.backward(dscore)

        return dh
```

정답 (긍정적 예)부분 하나와 NEG를 한 부분 여러개를 따로따로 계산한 다음에 은닉층에서 역전파 할때 다시 하나로 합쳐서 계산한다.

### CBOW

이제 파트별로 정리를 했으니 전체 모델을 구현해보자. CBOW모델의 구조를 살펴보면 in_layer 들이 window_size*2개 (context 개수) 만큼 있고 하나의 out_layer가 있다. 그러므로 먼저 다음과 같이 초기화를 해야 된다.

```python
W_in = 0.01 * np.random.randn(V, H).astype('f')
W_out = 0.01 * np.random.randn(V, H).astype('f')

self.in_layers = []
for i in range(2 * window_size):
    layer = Embedding(W_in)  
    self.in_layers.append(layer)
self.ns_loss = NegativeSamplingLoss(W_out, corpus, power=0.75, sample_size=5)
```

그리고 모델에 직접 파라미터하고 그라디언트 접근을 해야되고 최종적으로 W_in이라는 결과도 뽑아 내야 하므로 나머지 까지 다 정리하면 __init__은 아래와 같이 표현할 수 있다.

```python
class CBOW:
    def __init__(self, vocab_size, hidden_size, window_size, corpus):
        V, H = vocab_size, hidden_size

        W_in = 0.01 * np.random.randn(V, H).astype('f')
        W_out = 0.01 * np.random.randn(V, H).astype('f')

        self.in_layers = []
        for i in range(2 * window_size):
            layer = Embedding(W_in)  
            self.in_layers.append(layer)
        self.ns_loss = NegativeSamplingLoss(W_out, corpus, power=0.75, sample_size=5)

        layers = self.in_layers + [self.ns_loss]
        self.params, self.grads = [], []
        for layer in layers:
            self.params += layer.params
            self.grads += layer.grads

        self.word_vecs = W_in
```

CBOW모델은 in_layers에서 얻어온 값들을 다 합한 다음에 평균을 내고 negative sampling loss를 진행한다. forward의 parameter로 context와 target을 뽑았다면 context단어 별로 임베딩을 추합한 다음에 앞에 말했던것과 같이 진행하면 된다. 아래는 최종적인 코드이다.

```python
class CBOW:
    def __init__(self, vocab_size, hidden_size, window_size, corpus):
        V, H = vocab_size, hidden_size

        W_in = 0.01 * np.random.randn(V, H).astype('f')
        W_out = 0.01 * np.random.randn(V, H).astype('f')

        self.in_layers = []
        for i in range(2 * window_size):
            layer = Embedding(W_in)  
            self.in_layers.append(layer)
        self.ns_loss = NegativeSamplingLoss(W_out, corpus, power=0.75, sample_size=5)

        layers = self.in_layers + [self.ns_loss]
        self.params, self.grads = [], []
        for layer in layers:
            self.params += layer.params
            self.grads += layer.grads

        self.word_vecs = W_in

    def forward(self, contexts, target):
        h = 0
        for i, layer in enumerate(self.in_layers):
            h += layer.forward(contexts[:, i])
        h *= 1 / len(self.in_layers)
        loss = self.ns_loss.forward(h, target)
        return loss

    def backward(self, dout=1):
        dout = self.ns_loss.backward(dout)
        dout *= 1 / len(self.in_layers)
        for layer in self.in_layers:
            layer.backward(dout)
        return None
```

### SkipGram

Cbow 모델을 한번 구현해 봤으면 Skip-Gram은 레이어 개수를 앞 뒤로 바꾼것이기 때문에 구현하기가 쉽다. CBOW와 달리 SkipGram은 입력은 target으로 하고 출력은 context쪽을 한다는 것만 신경 쓰면 된다.

```python
class SkipGram:
    def __init__(self, vocab_size, hidden_size, window_size, corpus):
        V, H = vocab_size, hidden_size

        W_in = 0.01 * np.random.randn(V, H).astype('f')
        W_out = 0.01 * np.random.randn(V, H).astype('f')

        self.in_layer = Embedding(W_in)  
        self.loss_layers=[]
        for i in range(2 * window_size):
          layer = NegativeSamplingLoss(W_out, corpus, power=0.75, sample_size=5)
          self.loss_layers.append(layer)

        layers = [self.in_layer] + self.loss_layers
        self.params, self.grads = [], []
        for layer in layers:
            self.params += layer.params
            self.grads += layer.grads

        self.word_vecs = W_in

    def forward(self, contexts, target):
        h = self.in_layer.forward(target)

        loss=0
        for i, layer in enumerate(self.loss_layers):
            loss += layer.forward(h,contexts[:, i])
        return loss

    def backward(self, dout=1):
        dh=0
        for layer in self.loss_layers:
            dh+=layer.backward(dout)
        self.in_layer.backward(dh)
        return None
```

## Pytorch 구현

numpy로 구현을 했지만 GPU를 못 쓴다는 한계로 pytorch로 다시 구현을 할 것이다. 다소 까다로운 파이프라인 용어들이 많이 있어서 pytorch로 바꾸는데 좀 많이 애먹었다.

Embedding은 pytorch에서 기능을 제공해준다. *nn.Embedding* 그러므로 Embedding과 관련된 클래스들은 빼고 나머지 클래스를 잘 번역해 주면 된다.

### Negative Sampling

UnigramSampler에서는 초기화 부분에서는 self.word_p=word_p 대신 register_buffer메소드를 썼다. 이걸 쓰면 다음과 같은 이점이 있다고 한다.

- device상태 (CUDA) 자동이동
- state_dict 저장
- requires_grads=False 로 지정

그리고 이전 부분에 대해서 corpus에서 하나하나 새면서 확률 분포를 만들었지만 bincount함수를 이용해서 따로 만들었다. 기존에는 미니배치크기로 for문을 돌려서 사용했지만 gpu를 사용하는 pytorch에서는 적절치 않다고 생각했다. 

*get_negative_sample* 부분을 좀 손을 봤다. 앞서 설명했던 for문의 단점과 같은 이유이다.

```python
class UnigramSampler(nn.Module):
    def __init__(self, corpus, vocab_size, power, sample_size, device='cpu'):
        super().__init__()
        self.sample_size = sample_size
        self.vocab_size = vocab_size   

        corpus_t = torch.tensor(corpus, dtype=torch.long)
        word_p = torch.bincount(corpus_t, minlength=vocab_size).float()

        word_p = torch.pow(word_p, power)
        word_p /= torch.sum(word_p)
        self.register_buffer('word_p', word_p)

    def get_negative_sample(self, target):
        batch_size = target.shape[0]
        p = self.word_p.unsqueeze(0).expand(batch_size, -1).clone()  # (B, V)
        p.scatter_(1, target.unsqueeze(1), 0)                         # 각 행에서 target 위치만 0으로
        p = p / p.sum(dim=1, keepdim=True)
        return torch.multinomial(p, self.sample_size, replacement=False)
```

pytorch파이프라인 용어 중에서 처음 안 게 있어서 get_negative_sample을 자세히 공부해 보았다. self.word_p는 원래 차원이 (V,)였다. 그래서 차원을 더 늘릴려고 0부분에 unsqueeze를 하고 그걸 batch_size만큼 늘렸다. 이때, print(self.word_p.unsqueeze(0).expand(batch_size, -1))을 하면 우리가 원하는 값이 제대로 된것 처럼 보인다. 하지만 왜 .clone()을 붙여서 각각 복사시켰을까? 저걸 print하면 값이 제대로 나오는 것 처럼 보이지만 사실 같은 메모리를 빙글뱅글 돌며 값을 출력하는 것이다. 즉, 깊은 복사를 위해 새로운 메모리에 값을 심어야 되므로 .clone()을 쓴 것이다. 

그 다음은 scatter_인데 관례상 '_'는 in_place를 의미한다고 한다. 암튼 dim=1 (V) 방향에 대해서 값을 바꿀것이며,target은 차원이 (B,)이므로 1부분에 대해 unsqueeze를 해주고(B,1), 바꾸는 값은 0으로 바꾼다.

keepdim=True를 해줘야 (B,1) 브로드 캐스팅이 원할하게 된다고 한다.

torch.multinomial(input, num_samples, replacement) 은 input을 확률 변수로 두고 num_samples수 만큼 뽑는 것이다. replacement는 중복허용 여부이다.

NegativeSamplingLoss는 많이 깔끔해진 것을 볼 수 있다. out_embed는 외부에 있는 embedding을 가져온 것이다. (W_out을 CBOW나 SkipGram클래스에 대해 각각 선언했다.) 

```python
class NegativeSamplingLoss(nn.Module):
    def __init__(self, out_embed, corpus, vocab_size, power=0.75, sample_size=5,device='cpu'):
        super().__init__()
        self.sampler = UnigramSampler(corpus, vocab_size, power, sample_size, device)
        self.out_embed = out_embed


    def forward(self, h, target):
        batch_size = target.shape[0]
        negative_sample = self.sampler.get_negative_sample(target)

        pos_score = (self.out_embed(target) * h).sum(-1)        # (B,)
        neg_score = torch.bmm(self.out_embed(negative_sample), h.unsqueeze(2)).squeeze(2)  # (B, K)

        loss = F.logsigmoid(pos_score).mean() + F.logsigmoid(-neg_score).sum(-1).mean()

        return -loss
```

pos_score부분이나 loss부분은 똑같고 neg_score부분에서 for문을 제거했다. bmm은 미니배치상태일때 matrix multiplication에 쓰이는 용도이다. 실례로 위에 차원은 각각 (B,K,D),(B,D,1) (K=negative sample 수, D는 embedding 차원) 이다. 처음 알게된건데 sum(-1)은 마지막 부분 합하라는 의미라고 한다.

### CBOW

CBOW모델이 매우 깔끔해진 것을 볼 수 있다. 

```python
class CBOW(nn.Module):
    def __init__(self, vocab_size, embed_dim, corpus, power=0.75, sample_size=5,device='cpu'):
        super().__init__()
        self.in_embed = nn.Embedding(vocab_size, embed_dim)
        self.out_embed = nn.Embedding(vocab_size, embed_dim)

        with torch.no_grad():
            self.in_embed.weight.copy_(0.01 * torch.randn(vocab_size, embed_dim))
            self.out_embed.weight.copy_(0.01 * torch.randn(vocab_size, embed_dim))

        self.neg=NegativeSamplingLoss(self.out_embed, corpus, vocab_size, power, sample_size,device)

    def forward(self, context, target):
        # context: (B, C), target: (B,)
        h = self.in_embed(context).mean(dim=1)                      # (B, D)
        loss = self.neg(h, target)
        return loss
```

### SkipGram

```python
class SkipGram(nn.Module):
  def __init__(self, vocab_size, embed_dim, corpus, power=0.75, sample_size=5,device='cpu'):
    super().__init__()
    self.in_embed = nn.Embedding(vocab_size, embed_dim)
    self.out_embed = nn.Embedding(vocab_size, embed_dim)

    with torch.no_grad():
      self.in_embed.weight.copy_(0.01 * torch.randn(vocab_size, embed_dim))
      self.out_embed.weight.copy_(0.01 * torch.randn(vocab_size, embed_dim))

    self.neg=NegativeSamplingLoss(self.out_embed, corpus, vocab_size, power, sample_size,device)

  def forward(self, context, target):
    B, C = context.shape

    h = self.in_embed(target)                           # (B, D)
    h_expand = h.unsqueeze(1).expand(-1, C, -1)         # (B, C, D)
    h_flat = h_expand.reshape(B * C, -1)                # (B*C, D)
    context_flat = context.reshape(B * C)               # (B*C,)

    loss = self.neg(h_flat, context_flat)               # 기존 NegativeSamplingLoss 그대로 재사용
    return loss
```

SkipGram모델에서 for문을 지우기 위해 좀 애를 먹었다. 여기도 위와 같이 expand만을 하면 메모리의 깊은 복사가 되지 않는다. 그래서 h_flat으로 concatenate된 형태로 만들 때 reshape를 하면서 값을 깊게 복사할 수 있다. context도 아예 차원을 축소시켜버렸다. loss 입장에서 보면 SkipGram모델은 CBOW모델에서 context개수 만큼 계산을 배로 처리하는 것이므로 아예 flatten하게 만들어 버린 것이다. SkipGram이 원래 C배 많은 (target, context) 쌍에 대해 각각 loss를 계산해야 하는 모델이기 때문이다.

### Subsampling of frequent Words

앞서 설명했던 subsample을 할 것이다. 먼저 단어가 삭제될 확률을 아래의 식과 같이 정의를 해 줄 것이다.

$$f(w_i) = 1 - \sqrt{\frac{t}{P(w_i)}}$$

Unigram함수 처럼 단어 빈도를 먼저 정의해준 다음에 단어 마다 확률을 정의할 것이다. 이때 한 단어에서 P가 t(threshold=1e-5) 보다 낮을 경우는 1로 처리한다. 

```python
def negative_subsampling(corpus, vocab_size, threshold=1e-5, device='cpu'):
   
    corpus_t = torch.tensor(corpus, dtype=torch.long)
    word_p = torch.bincount(corpus_t, minlength=vocab_size).float()

    word_p /= torch.sum(word_p)   

    discard_prob = 1 - torch.sqrt(threshold / word_p)
    discard_prob = torch.clamp(discard_prob, min=0.0)   # 음수 방지 (희귀 단어는 버릴 확률 0)

    return discard_prob
```

그리고 매 에포크마다 subsample된 data를 만들어야 된다. 그러나 위 함수는 모든 에포크마다 공유하는 확률 분포를 나타내므로 분리시켜서 표현할 필요가 있다. 먼저 아래와 같이 어떤 단어를 버릴지 결정하는 함수 부터 만들어 보자.

```python
def subsample_corpus(corpus, discard_prob):
    corpus_t = torch.tensor(corpus, dtype=torch.long)
    p_discard = discard_prob[corpus_t]      # 각 위치 단어의 버릴 확률
    keep_mask = torch.rand(len(corpus)) >= p_discard
    return corpus_t[keep_mask].tolist()
```

무작위 난수를 통해 같은 확률을 나타내도 매 에포크마다 corpus는 다르다. 이제 위 corpus를 토대로 context와 target을 만들면 되는 것이다. 다음 객체도 영상을 참고하였지만 window_size 등과 같이 내가 조금 변형을 가했다.

```python
class Text8Corpus:
  def __init__(self,subsample_threshold=1e-5):
    if not os.path.exists('text8'):
        urllib.request.urlretrieve('http://mattmahoney.net/dc/text8.zip', 'text8.zip')
        with zipfile.ZipFile('text8.zip') as z:
            z.extractall()

    with open('text8', 'r') as f:
        raw_text = f.read()

    tokenized_text = raw_text.split()

    vocab = list(set(tokenized_text))
    self.word_to_id = {word: i for i, word in enumerate(vocab)}
    self.id_to_word = {i: word for i, word in enumerate(vocab)}

    self.corpus = [self.word_to_id[word] for word in tokenized_text]   
    self.vocab_size = len(vocab)

    self.discard_prob = negative_subsampling(self.corpus, self.vocab_size, threshold=subsample_threshold)

  def get_data(self, window_size):
    corpus_subsampled = subsample_corpus(self.corpus, self.discard_prob)
    data = []
    for i in range(window_size, len(corpus_subsampled) - window_size):
        context_ids = [corpus_subsampled[i + j] for j in range(-window_size, window_size + 1) if j != 0]
        target_id = corpus_subsampled[i]
        data.append((context_ids, target_id))

    return data,corpus_subsampled
```

## 실습

드디어 실습이다. 모델 특성상 train set밖에 없기 때문에 다른 모델보다 학습시키는 루프를 만들때는 매우 쉽게 만들 수 있었다. 

걸리는 부분이 하나 있었는데 매 에포크 마다 시간이 1시간이 소요되는 것을 보았다. 시간이 너무 오래걸리는거 같아서 원인이 무엇인지 찾아 보니 UnigramSampler에 있었다.

```python
def get_negative_sample(self, target):

        p = self.word_p.unsqueeze(0).expand(batch_size, -1).clone()  # (B, V)

        p = p / p.sum(dim=1, keepdim=True)
        return torch.multinomial(p, self.sample_size, replacement=False)
```

text8의 vocabsize는 20만개 정도 되는데 **위 부분에서 clone()을 할때 20만개 에다가 미니배치 만큼 곱하는 것 만큼 복사해야 되는 점**이 문제였다. 여기서 병목현상이 일어나서 시간을 많이 잡아먹은 것이다. 그래서 어차피 단어 종류 개수가 20만개니 negative sampling 할 때 target이 뽑힐 확률은 매우 적을 것이므로 target의 단어 확률분포를 0으로 만드는 것이 아닌 그대로 두는것이 좋겠다고 생각했다. 그래서 다음과 같이 코드를 변형하였다.

```python
def get_negative_sample(self, target):
        batch_size = target.shape[0]
        # 어차피 target 뽑힐 확률 1/200000
        neg = torch.multinomial(
            self.word_p, batch_size * self.sample_size, replacement=True
        )
        return neg.view(batch_size, self.sample_size)
```
