## Intrduction

Alexnet을 만들때 해결하고자 하는 문제점은 **Larger Training Set**이었다. 세상에는 너무나 다양한 것들이 많은데 당시 모델로는 이를 해결하기 힘들었다. ILSVRC라고 컨테스트가 있는데 대용량 데이터셋인 **ImageNet**을 학습시키는 것이 관건이었는데 GPU 2장으로 이걸 뚫어 버렸다. 딥러닝의 가능성을 열어준 논문이라 보면 된다.

## Architecture

5개의 Convolutional 층과 3개의 Fully-connected층으로 구성되어있다. 그리고 다음 4개는 AlexNet의 성능을 높여준 대표적인 4가지 요소들이다. 중요한 요소일 수록 앞에 배치했다.


### ReLU

![Alexnet1](assets/img/alexnet1.png)

기존에는 활성화 함수를 시그모이드 함수나 탄젠트하이퍼볼릭(tanh(x))를 썼다고 한다. 근데 ReLU를 쓰니 **Training Time**이 훨씬 줄었다고 한다. 

### GPU사용

GPU하나 사용하기에는 크기가 너무 커서 2개를 써서 했다고 한다. 논문 구현 파트에서 보겠지만 각 레이어가 두 갈래로 나눠진 것을 볼 수 있을 것이다. 그러나 **특정레이어에서 GPU들 끼리 서로 데이터를 주거니 받거니** 하다가 다시 따로따로 레이어에서 각자 학습한다. 따로 하는것이 cross-validation문제가 된다고하긴 하지만 당시에는 최선의 선택이었다. 지금은 두 갈래로 안 나누고 하나로 합쳐서 학습시켜도 된다. 실제로 대부분 딥러닝 교재에서는 그냥 하나로 합쳐버린다. 하지만 이 블로그는 그냥 논문 그대로 구현했다. (물론 GPU 2장을 각각 코드와 연동하는 작업은 없다)

### Local Response Normalization

ReLU가 좋긴 한데 normalization기능은 없다. batch-normalization에서 말했다 시피 normalization쓰는 이유는 보통 분포를 원하는 방향으로 변형시키기 위해서이다. 

$a^i_{x,y}$는 입력값이고 i는 kernel(채널)번호, (x,y)는 이미지 위치이고 $b$를 결과라고 했을때 Local Respong Normalization(LRN)에 대한 공식은 아래와 같다.

$$
b^i_{x,y} = a^i_{x,y} / \left( k + \alpha \sum_{j=\max(0,i-n/2)}^{\min(N-1,i+n/2)} (a^j_{x,y})^2 \right)^\beta
$$

$N$은 모든 채널의 수이다. (배치 사이즈 수 아님) 나머지 $n=5$,$\alpha=10^{-4}$,$\beta=0.75$,$k=2$는 hyper-parameter이다. 이 논문에서 별도의 튜닝은 하진 않는다. ReLU활성화 함수를 쓰고 LRN을 쓴다고 한다. 코드는 아래와 같다. (cp는 numpy와 같은 역할을 하는 cupy이다.)

```python
def forward(self,x):
  N,C,H,W=x.shape
  self.x=x
  self.scale=cp.zeros_like(x)

  for i in range(C):
      ch_start = max(0, i - self.n // 2)
      ch_end = min(C, i + self.n // 2 + 1)
      self.scale[:, i, :, :] = self.k + self.alpha * cp.sum(
          x[:, ch_start:ch_end, :, :] ** 2, axis=1
      )

  out = x / (self.scale ** self.beta)
  return out
```

LRN역전파는 $i=j$일때와 그렇지 않을때로 구분해야한다. $D^i = k + \alpha \sum_{j=\max(0,i-n/2)}^{\min(N-1,i+n/2)} (a^j_{x,y})^2$ 라고 가정하자.

- $i=j$

$$
\frac{\partial b^i}{\partial a^i} = (D^i)^{-\beta} + a^i \cdot (-\beta)(D^i)^{-\beta-1} \cdot 2\alpha a^i
$$

$$
= (D^i)^{-\beta} - 2\alpha\beta (a^i)^2 (D^i)^{-\beta-1}
$$

- $i \neq j$ (즉 $i \in [\max(0, j-n/2),\ \min(N-1, j+n/2)]$ 인 경우)

$$
\frac{\partial b^j}{\partial a^i} = a^j \cdot (-\beta)(D^j)^{-\beta-1} \cdot 2\alpha a^i
$$

$$
= -2\alpha\beta a^i a^j (D^j)^{-\beta-1}
$$

이걸 합치면...

$$
\frac{\partial L}{\partial a^i} = \frac{\partial L}{\partial b^i} \cdot \frac{\partial b^i}{\partial a^i} + \sum_{j \neq i} \frac{\partial L}{\partial b^j} \cdot \frac{\partial b^j}{\partial a^i}
$$

$$
= \frac{\partial L}{\partial b^i}(D^i)^{-\beta} - 2\alpha\beta \sum_{j} \frac{\partial L}{\partial b^j} a^j (D^j)^{-\beta-1} a^i
$$

코드는 아래와 같다.

```python
def backward(self, dout):
  x = self.x
  N, C, H, W = x.shape
  dx = cp.zeros_like(x)

  for i in range(C):
      ch_start = max(0, i - self.n // 2)
      ch_end = min(C, i + self.n // 2 + 1)
      for j in range(ch_start, ch_end):
          dx[:, j, :, :] += dout[:, i, :, :] * \
              (-self.beta * (self.scale[:, i, :, :] ** (-self.beta - 1))) * \
              (2 * self.alpha * x[:, j, :, :])

  dx += dout * (self.scale ** (-self.beta))
  return dx
```

i=j(아래)일때와 그렇지 않을때(위)를 구분해서 적은것을 볼 수 있다.

### Pooling 겹치게 하기

stride($s$) 수랑 한 커널(neighborhood)사이즈 ($z$)를 같게 해놓는 경우가 일반적이다. 예를 들어 입력이 4*4사이즈면 보통은 스트라이드 2 에다가 이웃 사이즈가 2이다. 그러나 여기서는 $s<z$로 처리를 했다. overfitting이 덜 된다고 한다. 풀링 결과가 서로 독립적이지 않아서 특징들 사이의 **상관관계**가 잘 보존되기 때문인 것 같다.

## Data Augmentation (전처리)

후반부에는 Overfitting 을 억제하기 위해 씨름(combat)하고 있는 내용이다. 하나는 **DropOut**으로 이미 Architecture안에 Fully-conncted부분에 구현이 되어있다. 그리고 나머지 하나는 **데이터 전처리(Data Augmentation)** 이다. 전처리 방법엔 크게 2가지를 제시한다.

- 이미지 변형 및 수평 반전

256개의 이미지에서 224 x 224 크기의 패치를 추출 하는 방법을 제안한다. 데이터셋의 크기가 2048배가 늘어난다고 하긴 하지만 안 하면 overfitting 돼서 층을 더 얕게 해야 된다고 했다.

테스트 시에는 256 x 256 이미지에서 224 x 224 패치를 뽑는데 좌측 상단, 좌측 하단, 중앙, 우측 상단, 우측 하단 이렇게 5개를 뽑는다. 그리고 수평 반전도 한다. 총 10개의 패치를 추출하여 훈련한다. 10개의 패치에 대해 평균을 낸다.

현실적으로 2048배는 개오바고 테스트시에도 평균내는 일은 매우 번거롭기 때문에 데이터셋 중 무작위로 크롭하는 것과 수평반전하는 방법을 선택했다. 아래는 데이터셋을 전처리 하는 코드이다.

```python
transform_train = transforms.Compose([
    transforms.Resize((256, 256)),
    transforms.RandomCrop(224),
    transforms.RandomHorizontalFlip(),
    transforms.ToTensor(),
])
```

256x256 이미지로 조정시킨 다음이 224 사이즈 이미지로 크롭을 한다. 그리고 데이터 셋 중 무작위로 수평 반전을 시킨다.

```python

    transforms.Resize((256, 256)),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
])
```

test 케이스는 center crop만 진행했다.

- RGB 채널 강도 바꾸기

training set 전체의 RGB 픽셀값에 대해 PCA를 수행한다. 
eigen vector (**p**)에 평균이 0이고 표준편차가 0.1인 가우시안 분포에서 나온 난수($\alpha$)를 곱한 eigen value(\lambda) 값을 내적해서 노이즈를 만든다.
막무가내로 만든 노이즈가 아니라 특징값을 추출해서 만든 노이즈이므로 학습에 도움이 된다. 식으로 표현하면 아래와 같다.

$$
\begin{bmatrix} I^R_{xy} \\ I^G_{xy} \\ I^B_{xy} \end{bmatrix} \mathrel{+}= [\mathbf{p}_1, \mathbf{p}_2, \mathbf{p}_3][\alpha_1\lambda_1, \alpha_2\lambda_2, \alpha_3\lambda_3]^T
$$

코드로 보면 아래와 같이 표현할 수 있다. 외부에서 PCA를 수행해서 eigenvector,eigenvalue 값을 추출하고 노이즈 추가를 진행한다. 아래 클래스는 claude 코드와 같이 짰다.

```python
class PCAColorAugmentation:
  def __init__(self, eigenvectors, eigenvalues, std=0.1):
      self.eigenvectors = eigenvectors  # 미리 계산된 값
      self.eigenvalues = eigenvalues
      self.std = std

  def __call__(self, img):
      img = img.permute(1, 2, 0)
      alphas = torch.normal(0, self.std, size=(3,))
      delta = self.eigenvectors @ (alphas * self.eigenvalues)
      img = (img + delta).clamp(0, 1)
      return img.permute(2, 0, 1)
```

```python
transform_train = transforms.Compose([
    transforms.Resize((256, 256)),
    transforms.RandomCrop(224),
    transforms.RandomHorizontalFlip(),
    transforms.ToTensor(),
    PCAColorAugmentation(std=0.1),
])
```
