## Introduction

![cnn1](assets/img/cnn1.png)

CNN의 시초는 Hubel과 Wiesel의 1959년에 이루어진 고양이 실험에서 부터 시작한다. 위의 그림처럼 고양이에게 시각정보가 주어졌을 때 고양이의 뉴런반응을 통해 어떻게 관찰하는지에 대해서 연구하는 내용이다. 이미지에 대해 전체부분이 아니라 **특정한 부분에서 뉴런의 반응이 온다는 것**을 관찰할 수가 있다. 이런 **국보수용영역(local receptive field)** 들이 겹쳐져서 전체적인 이미지에 대해 인식을 하게 됨을 증명하는 시험이었다. 그리고 이런 아이디어를 빌려 만든 network가 바로 CNN이다. (CNN 말고도 상당히 많은 이미지 네트워크가 이를 토대로 이용하긴 한다.) 이를 통해 모서리(edge), 끝점(end-point), 가장자리(corner) 등을 추출할 수 있다. 

## Implementation

CNN에 어떻게 계산 되는지 구체적으로 적기 보단 내가 코드 구현하면서 코드를 이게 무슨 의미인지를 파악하면서 학습하는 것으로 중점을 두었다.

### Convolution

![cnn2](assets/img/cnn2.png)

인풋 데이터에서 그대로 CNN연산을 하면 시간 복잡도가 레전드이다. 따라서 메모리를 조금 희생하더라도 입력데이터에서 커널(필터)에서 연산할 모든 부분을 뽑아 2차원 형태의 array로 만드는 것이 효율적이다. 이 방법을 im2col이라고 한다. 위 그림을 참고하면 이해하기가 쉽다.

입력데이터에서 커널의 사이즈에 맞게 뽑은 배열의 수는 곧 특징맵(feature map)에서 채널 하나당 픽셀 개수와 똑같다. 특징맵의 가로와 세로를 각각 out_w, out_h라고 할 때 커널 사이즈에 맞게 뽑은 배열의 수는 out_h*out_w라고 볼 수 있다. 그러므로 사전에 아래와 같은 공식에 대입해 먼저 im2col의 행에 대한 크기를 구할 수 있다.

$$
O = \left\lfloor \frac{I - K + 2P}{S} \right\rfloor + 1
$$

I : input size K : kernel size P : padding S : stride

다음으로 필터(커널)에 대한 것이다. 필터의 개수는 곧 출력 데이터의 채널 수이다. 그리고 당연한 얘기지만 하나의 필터에 대해서 parameter의 수는 im2col을 씌운 입력데이터에서 열의 크기와 같다.

그 다음에는 bias로 편향값을 브로드캐스트해서 더하면 끝난다.

코드로 구현하면 아래와 같이 구현할 수 있다.

```python
def im2col(input_data, filter_h,filter_w, stride=1, pad=0):
    N, C, H, W = input_data.shape
    out_h=(H+2*pad-filter_h)//stride+1
    out_w=(W+2*pad-filter_w)//stride+1

    img=cp.pad(input_data, [(0,0),(0,0),(pad,pad),(pad,pad)], 'constant')
    col=cp.zeros((N,C,filter_h,filter_w,out_h,out_w))

    for y in range(filter_h):
      y_max=y+stride*out_h
      for x in range(filter_w):
        x_max=x+stride*out_w
        col[:, :, y, x, :, :] = img[:, :, y:y_max:stride, x:x_max:stride]

    col=col.transpose(0,4,5,1,2,3).reshape(N*out_h*out_w,-1)
    return col
```

참고로 cp는 cupy이다. 원래 numpy하고 np랑 같다고 보면 된다. 굳이굳이 gpu써보고 싶다고... 그렇게 됐다.

```python
cp.pad(input_data, [(0,0),(0,0),(pad,pad),(pad,pad)], 'constant')
```

(x,x)꼴이 있는데 각각 앞으로 뒤로 값을 늘려주는 것이다. 패딩(padding)이다 말그대로.

```python
col[:, :, y, x, :, :] = img[:, :, y:y_max:stride, x:x_max:stride]
```

처음에 이 코드를 햇갈렸었는데 미니배치하고 채널을 무시하면 아래와 같이 볼 수 있다.

```python
col[y, x, :, :] = img[y:y_max:stride, x:x_max:stride]
```

브로드 캐스트를 빼고 보면 위와 같이 표현할 수 있다. 끝에 2개가 특징맵의 사이즈인 것을 고려하면 이해하기 수월해진다.

다음은 최종적으로 출력하는 식이다.

```python
def forward(self,x):
    FN,C,FH,FW=self.W.shape
    N,C,H,W=x.shape
    out_h=1+int((H+2*self.pad-FH)/self.stride)
    out_w=1+int((W+2*self.pad-FW)/self.stride)

    col=im2col(x,FH,FW,self.stride,self.pad)
    col_W=self.W.reshape(FN,-1).T

    out=col@col_W+self.b
    out=out.reshape(N,out_h,out_w,-1).transpose(0,3,1,2)

    self.x = x
    self.col = col
    self.col_W = col_W

    return out
```

col_w나 out은 reshape로 끝나는 것을 볼 수 있다.

다음은 역전파를 해야 되는데 복잡하니 차근차근 생각해보았다. 일단 연산을 일반 과정이 아닌 reshape로 변형시켜서 했으니 역전파 할 때도 변형시킬 필요가 있다.
```python
dout=dout.transpose(0,2,3,1).reshape(-1,FN)
```

bias 값에 대한 역전파는 다음과 같이 나타낼 수 있다.

$$
\frac{\partial L}{\partial b_{c,h,w}} = \sum_{n} \frac{\partial L}{\partial Y_{n,c,h,w}}
$$

bias가 채널수로 밖에 안 되어있는 1차원인줄 알았는데 픽셀값까지 고려한 편향도 있다고 한다. (오...)

```python
self.db=cp.sum(dout,axis=0)
```

W를 가중치라 하고 X를 im2col을 씌운 입력데이터라 할때 W에 대한 그라디언트는 아래와 같이 표현이 가능하다.

$$
\frac{\partial L}{\partial W} = X^T \cdot \frac{\partial L}{\partial Y}
$$

내적을 하였으면 원래 필터의 형태로 다시 돌려놓는 작업도 필요하다.]

```python
self.dW=self.col.T @ dout
self.dW=self.dW.transpose(1,0).reshape(FN,C,FH,FW)
```

이제 입력데이터를 구해야될 차례인데 마찬가지로 아래와 같이 표현할 수 있다.

$$
\frac{\partial L}{\partial X} = \frac{\partial L}{\partial Y} \cdot W^T
$$

이때도 X를 원래 형태로 바꿔야 하는데 col2im을 쓸 것이다.

```python
dcol=dout@self.col_W.T
dx=col2im(dcol,self.x.shape,FH,FW,self.stride,self.pad)
```

여기서 좀 머리가 아파 보이지만 im2col을 이미 구현했으면 생각보다 쉽게 구현이 가능하다.

```python
img = cp.zeros((N, C, H + 2*pad + stride - 1, W + 2*pad + stride - 1))
for y in range(filter_h):
  y_max=y+stride*out_h
  for x in range(filter_w):
    x_max=x+stride*out_w
    img[:, :, y:y_max:stride, x:x_max:stride] += col[:, :, y, x, :, :]

return img[:, :, pad:H + pad, pad:W + pad]
```

img(원래 형태)에서 $I + 2P + S - 1$형태는 img에서 범위 최댓값을 의미한다.
그리고 최종미분값은 가능한 그라디언트값을 모두 더한 것이므로 브로드 캐스트를 빼면 다음과 같이 표현 가능하다.

```python
img[y:y_max:stride, x:x_max:stride] += col[y, x, :, :]
```

그리고 return 할때는 padding범위를 빼주면 입력데이터에 대한 원래 형태로 복구 시킬 수 있다.

### MaxPooling

Pooling할때도 원래형태보다는 im2col로 분해해서 하는게 편하다. im2col을 호출할때 2번째 3번째 사이즈를 곱한것이 im2col 변형 후 열의 크기라 보면 된다. 그리고 각 행에서 최댓값을 뽑을 수 있도록 변형시킨다. 코드로 나타내면 아래와 같다.

```python
col = im2col(x, self.pool_h, self.pool_w, self.stride, self.pad)
col = col.reshape(-1, self.pool_h*self.pool_w)

out = cp.max(col, axis=1)
out=out.reshape(N,out_h,out_w,C).transpose(0,3,1,2)
```

그러나 우리는 역전파라는 것을 해야된다. 일단 그렇게 할려면 다시 out값을 변형 시켜야 된다.

```python
dout=dout.transpose(0,2,3,1)

pool_size=self.pool_h*self.pool_w
dmax=cp.zeros((dout.size,pool_size)) #(N*oh*ow*C, pool_size)
```

아 근데 forward에서 행마다 최댓값의 인덱스값을 알아야 한다. 다른 곳에서는 역전파가 안 일어나니깐... 그러므로 forward함수에서 아래와 같이 수정한다.

```python
arg_max = cp.argmax(col, axis=1)
out = cp.max(col, axis=1)
out=out.reshape(N,out_h,out_w,C).transpose(0,3,1,2)

self.x=x
self.arg_max=arg_max
```

그리고 다시 backward함수로 돌아가 이 줄을 추가할 수 있다.

```python
dmax[cp.arange(self.arg_max.size), self.arg_max.flatten()] = dout.flatten()
```

마지막으로 형태변환 까지 한다.

```python
dmax = dmax.reshape(dout.shape + (pool_size,)) #(N,oh,ow,c,pool_size)

dcol = dmax.reshape(dmax.shape[0] * dmax.shape[1] * dmax.shape[2], -1)
dx = col2im(dcol, self.x.shape, self.pool_h, self.pool_w, self.stride, self.pad)
```

이렇게 역전파를 끝낼 수 있다.
