## Probability

여기 부분은 모든 언어모델과 동일한 부분이다.

단어를 하나하나 생성할 때, 생성했던 언어들도 고려해야 한다. 현재 생성할려고 하는 word를 $w_t$라 하고 앞에 생성했던 두 단어를 바탕으로 생성하는 확률을 하면 다음과 같다.

$$P(w_t \mid w_{t-1}, w_{t-2})$$

위와 같은 확률로 표현할 수 있고 $w_1$ 부터 $w_m$ 까지 나올 수 있는 단어의 확률을 아래와 같이 표현할 수 있다.

$$P(w_1, \dots , w_{m})=\Pi_{t=1}^m P(w_t \mid w_1, \dots, w_{t-1})$$

뭐랑 형상이 비슷하지 않은가? 그렇다 likelihood가 떠올린다. 그러나 likelihood 처럼 최댓값을 구하기 위해 미분하거나 그럴필요는 없고 cross-entropy로 처리할 수 있다.

$$\sum_{t=1}^m \log P(w_t \mid w_1, \dots , w_{t-1})$$

즉, 우리가 모델을 설계하기 위해선 과거의 단어가 처리되었던 데이터를 끌고 오면 되는 것이다.

## RNN

단어를 tokenizer시키고 embedding 벡터로 만든 다음에 순차적으로 RNN에 넣는다. 실제로는 RNN이라는 객체는 하나지만 같은 가중치를 공유하며 여러개의 토큰을 집어넣는 형태로 코드가 짜진다. 그러므로 RNN을 다음과 같이 풀어쓴다.

![rnn1](assets/img/lstm1.png)

그림을 보면 잘 알겠지만 출력물을 다시 집어 넣는 것이 RNN의 핵심이다. 수식으로 풀면 다음과 같이 나타낼 수 있다.

$$h_t=\tanh(W_x x + W_h h_{t-1}+ b)$$

이 결과로 다음 timestep의 데이터로 쓰며 fully-connected로 통과 시켜 각각의 timestep에서 나온 결과에 대해 다음 timestep의 토큰의 확률로 손실함수를 구한다. 아래와 그림을 보면 이해가 편하다.

### BPTT 

단어 셋 전체로 학습을 시킨다면 역전파를 하는데 시간이고 정확성이고 문제가 많이 남을 것이다. 그래서 일정 부분을 잘라내어 iteration으로 한 다음에 그 부분을 학습 시키고 RNN에서 나온 출력물은 다음 iteration에서 쓰는 것을 제안한다. 

![rnn3](assets/img/lstm4.png)

이때, 다 객체를 다 펼쳐놓아서 이렇게 보이는거지 같은객체이기 때문에 실제 gradient는 한 이터레이션 모든 gradient를 추합해서 구한다.

## RNN 문제점

그러나 RNN에서는 큰 문제가 있다. 역전파로 학습하는데 gradient가 소실되거나 증폭 될 수도 있다는 것이다. tanh가 기울기를 계속 감소시킨다는 것도 있고 RNN출력할때 특정 부분이 gradient가 매우 높거나 낮은게 반복되면 아예 증폭해버리거나 감소될 수도 있다. 

증폭 문제는 **Gradient Clipping**이라는 기법으로 해결할 수 있다. $\hat{g}$를 **모든 매개변수의 기울기를 하나로 모은것**이라고 할 때  $\lVert \hat{g} \rVert$ 가 *threshold* ($\tau$)를 넘으면 모든 gradient에 대해 다음과 같이 표현할 수 있다.

$$g = g \frac{\tau}{\lVert \hat{g} \rVert}$$

pytorch에서는 역전파를 하고 난 뒤 다음 코드를 붙여주면 된다. ($\tau = 5$)

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=5)
```

## LSTM

그러나 기울기 소실 부분은 RNN구조상에서 발생하는 문제이기 때문에 아예 근본부터 뜯어 고쳐야 된다. 다음 그림은 LSTM 구조이다.

![lstm1](assets/img/lstm3.png)

그림에 $c_t$라고 **기억 셀**이 추가 되었다. 그래서 tanh로 인한 기울기 소실에 대해서 조치를 할 수 있게 되었다. 이제 다른 기울기 소실에 대해서 문제를 해결해야 하는데 이는 다음 4개의 요소로 해결할 수 있다.

- output

$$o=\sigma(x_t W_t^{(o)} + h_{t-1} W_h^{(o)} + b^{(o)})$$

$\sigma$는 sigmoid함수로 구간이 0~1.0 까지이다. 즉, output쪽에서 결과 h에 대해 데이터를 조절하겠다는 의미로 놓은 것이라 보면 된다.
  
- forget

$$f=\sigma(x_t W_t^{(f)} + h_{t-1} W_h^{(f)} + b^{(f)})$$

forget도 마찬가지이다. c의 벡터에서 어떤 부분의 특성을 상쇄시키는 것이다. 우리가 읽고 있는데 이제 어느 부분은 기억을 할 이유가 없는 단어들이 있듯이 LSTM에도 이런걸 추가 했다. 

- Input modulation (g)

$$g=\tanh(x_t W_t^{(g)} + h_{t-1} W_h^{(g)} + b^{(g)})$$

g는 f와 반대로 새로운 정보를 셀에 주입하는 것이다. where라는 것이 튀어나오면 주어가 튀어나올 확률이 높듯이 말이다. 이처럼 LSTM은 어휘의 특성을 더 잘 살린 아키텍쳐라고 볼 수 있다.

- input

$$i=\sigma(x_t W_t^{(i)} + h_{t-1} W_h^{(i)} + b^{(i)})$$

input은 새로 주입되는 정보의 g에 대해서 조절을 한다.

위와같이 LSTM은 기울기 상쇄효과와 더불어 어휘의 특성을 더 잘 살리는 모델이 되었다. 코드는 아래와 같다. pytorch로 책(밑바닥 부터 시작하는 딥러닝 2)을 참고해서 만들었다.

```python
class LSTM(nn.Module):
  def __init__(self, input_size, hidden_size):
    super(LSTM, self).__init__()
    self.H=hidden_size
    self.fc1=nn.Linear(input_size,4*hidden_size, bias=True)
    self.fc2=nn.Linear(hidden_size,4*hidden_size, bias=False)

  def forward(self,x,h_prev,c_prev):
    A=self.fc1(x)+self.fc2(h_prev)

    f=torch.sigmoid(A[:,:self.H])
    i=torch.sigmoid(A[:,self.H:2*self.H])
    o=torch.sigmoid(A[:,2*self.H:3*self.H])
    g=torch.tanh(A[:,3*self.H:])

    c=f*c_prev+i*g
    h=o*torch.tanh(c)

    return h,c
```

### LSTM 개선

기존의 LSTM 방식으로만 설정하면 overfitting 등과 같은 현상 때문에 perplexity가 낮게 나올 수도 있다. 그러므로 다음 2가지 방법을 쓰는것을 고려한다.

- Dropout

오버피팅이 일어나니 dropout을 쓰는건 매우 익숙한 일이다. 하지만 매 timestep마다 다른 mask를 쓰는 것 보다 같은 mask를 쓰는것이 더 학습에 좋다. 노드마다 계속 다른 정보를 저장할텐데 장기적으로 봤을땐 계속 다른 노이즈를 주입하게 되어 학습에 방해가 된다는 것이다. bayesian 관점에서 수치적으로 설명을 할 수 있지만 이 글은 굳이 깊게 들어가고 싶지 않기 때문에 [논문](https://arxiv.org/pdf/1512.05287)참고 하길 바란다.
  
- Weight tying

embedding층과 마지막 fully-connected층 사이의 가중치를 서로 공유하는 기법이다. 파라미터나 과적합 감소는 추론이 가능하지만 이렇게까지 하는 이유는 뭘까 하고 [논문](https://arxiv.org/pdf/1611.01462)을 찾아봤다. embedding은 단어 ID에서 하나의 은닉층으로 변환하는 거면 fully-connected는 그 반대이다. 즉, 서로 비슷한 성격의 파라미터를 공유하는거나 마찬가지이고 fully-connected를 학습할때 embedding을 같이 학습시키는 것은 의미가 있는 것이다. 더 디테일한건 논문을 보자.

## Architecture

LSTM도 BPTT구현을 하기 위해서 책에서는 TimeLSTM을 만들었고 나도 그렇게 했다. 하나의 iteration에 들어갈 단어의 수를 *seq_size*라고 지정했을때 객체를 하나 만들어서 for문으로 timestep마다 parameter를 집어 넣게 했다. 같은 parameter를 공유하며 다른 인수를 받는것이다. Dropout도 같은 mask (parameter)를 공유해야 하므로 미리 객체를 만들어 놨다.

```python
class TimeLSTM(nn.Module):
  def __init__(self, input_size,hidden_size, dropout=0.0):
    super(TimeLSTM, self).__init__()
    self.cell=LSTM(input_size, hidden_size)
    self.dropout_layer = nn.Dropout(p=dropout)
    self.hidden_size=hidden_size

    self.h, self.c= None, None

  def forward(self,xs,h0=None,c0=None):
    batch_size,seq_size,_=xs.shape

    if h0 is not None:
        self.h = h0
        self.c = c0
    elif self.h is None:
        self.h = xs.new_zeros(batch_size, self.hidden_size)
        self.c = xs.new_zeros(batch_size, self.hidden_size)

    ones = xs.new_ones(batch_size, self.hidden_size)
    mask = self.dropout_layer(ones)

    hs=xs.new_zeros(batch_size, seq_size, self.hidden_size)
    for t in range(seq_size):
      h_input = self.h * mask
      self.h, self.c = self.cell(xs[:,t,:],h_input,self.c)
      hs[:,t,:]=self.h
      # c 에서는 mask 쓰면 안 됨

    return hs

  def detach_state(self):
    if self.h is not None:
        self.h = self.h.detach()
        self.c = self.c.detach()

  def reset_state(self):
    self.h, self.c = None, None
```

이때, detach()는 역전파를 끊는 용도로 쓴다. 이전 이터레이션에서 계산했던 parameter들에 대하여 역전파를 진행하면 BPTT를 쓰는 의미가 없다. reset_state는 epoch돌 때마다 쓰면 된다.

나머지 Time레이어는 단순 pytorch메소드로 처리가 가능하다. 다 일일이 구현해야 될 줄 알았는데, 어차피 요구하는 차원 외에는 자동으로 차원을 유지하게 해줘서 아키텍쳐 짜는데 큰 힘이 들지 않았다. 조금 예외가 있다면 Timedropout에서 위에처럼 먼저 __init__에서 객체를 선언하고 mask를 만들어서 처리를 했다. 자세한건 [코드](https://github.com/Alchwalch/Deep-Learning-Study/blob/main/NLP/LSTM.ipynb)를 참고하면 될 것 같다.

전체를 아우르는 모델은 아래와 같다.

```python
class LSTMlm(nn.Module):
  def __init__(self,vocab_size,embedding_dim,hidden_size,num_layers=2,dropout=0.0):
    super(LSTMlm, self).__init__()
    self.embedding=TimeEmbedding(vocab_size,embedding_dim)
    self.time_dropout=TimeDropout(dropout)
    self.lstm=TimeLSTM(embedding_dim,hidden_size,dropout=0.0)

    layers=[TimeDropout(dropout)]
    for _ in range(num_layers-1):
      layers.append(TimeLSTM(hidden_size,hidden_size,dropout=0.0))
      layers.append(TimeDropout(dropout))

    self.lstm_layers=nn.Sequential(*layers)

    self.linear=TimeLinear(hidden_size,vocab_size)
    self.loss=TimeCrossEntropy()

    if embedding_dim==hidden_size:
        self.linear.linear.weight=self.embedding.embedding.weight

  def forward(self,xs,ts,h0=None,c0=None):
    xs=self.embedding(xs)
    xs=self.time_dropout(xs)
    xs=self.lstm(xs,h0,c0)
    xs=self.lstm_layers(xs)
    xs=self.linear(xs)
    return self.loss(xs,ts)

  def reset_state(self):
    for m in self.modules():
      if isinstance(m, TimeLSTM):
          m.reset_state()

  def detach_state(self):
    for m in self.modules():
      if isinstance(m, TimeLSTM):
          m.detach_state()
```

## 실습 

PTB 데이터셋으로 실습을 진행했다. 데이터를 seq_size단위로 묶어서 iteration을 진행한다는 것은 알지만 batch는 어떻게 처리하는지 궁금할 수 있다. PTB 데이터셋을 전보 tokenizer 시키고 총 단어 수에다가 batch_size만큼 나누면 된다.

```python
def batchify(data, batch_size):
    n_batch = data.size(0) // batch_size
    data = data[:n_batch * batch_size] # 마지막에 남은 자투리는 그냥 잘라버리기
    data = data.view(batch_size, -1).contiguous() #contiguous로 (tensor 기준에서) 메모리 접근 용이하게 함
    return data
```

그리고 iteration마다 batch마다 seq_size만큼 단어를 받게 만들면 된다.

```python
def get_batches(xs_b, ts_b, seq_size):
    total_len = xs_b.size(1)
    for i in range(0, total_len - 1, seq_size):
        end = min(i + seq_size, total_len)
        x_chunk = xs_b[:, i:end]   # (batch_size, chunk_len)
        t_chunk = ts_b[:, i:end]   # (batch_size, chunk_len)
        yield x_chunk, t_chunk
```
