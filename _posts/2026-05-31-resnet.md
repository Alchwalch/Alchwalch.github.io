## VGGNet

ResNet은 VGGNet의 문제점을 고치기 위해 만든 모델이라고 해도 과언이 아니다. 그러므로 먼저 VGGNet에 알아 볼 필요가 있다. 

![vggnet](assets/img/resnet1.png)

기존에 있었던 모델과 다르게 VGGNet에서 강조되는 것은 **깊이(Depth)** 이다. 기존의 모델에선 7x7 커널 사이즈로 돌려서 convolution을 하나로 끝냈다면 VGGNet은 **convolution layer을 3개**로 늘리고 **커널 사이즈를 3x3**으로 줄였다.
이렇게 층을 높이면 **비선형성을 처리하는 레이어**가 많아져 모델결과가 더 **discriminative**해지고 3x3으로 커널 사이즈를 줄여서 **parameter개수를 줄인다는** 이점이 있다. 그리고 매 Convolutional Layer마다 Maxpooling을 넣는게 아니라 3개 마다 하나씩 넣는 것을 볼 수가 있다. 이는 충분한 깊이의 feature 추출 후에만 해상도를 줄이자는 의미로 그만큼 VGGNet은 깊이에 집착을 한다.

## ResNet

### Problem

그러나 문제가 있다. VGGNet 같이 깊이가 깊은 모델들에서 **그라디언트가 사라지거나 폭주(Vanishing/exploding gradients)** 하는 일이 발생한 것이다. 이 문제는 주로 *초기화할때나 레이어에서 정규화(normalized initialization and normalization layer)* 로 문제를 해결하면 된다고 여겨져 왔다. 그러나 깊이를 늘리면 정확도가 포화되는 **degradation problem** 은 여전히 일어났다. 심지어 training error도 높아지는 것을 보아 overfitting의 문제는 아니다. 이론적으로 깊은 모델은 최소한 얕은 모델만큼은 나와야 한다. 추가된 레이어가 identity mapping($f(x)=x$)만 해도 결과가 같아야 하기 때문이다. 그러나 실제로는 오히려 더 나빠졌다.

### Deep Residual Learning

![resnet2](assets/img/resnet2.png)

이 문제를 다룰 때 기존에 일부 연속된 레이어들을 근본적으로(underlying)보지 않고 약간 추가적인 잔여물(residual)이라는 시각으로 보자고 제안한다. 쉽게 말하면 위 그림과 같이 들어오는것(**x**)와 두 레이어를 통과해서 나온 결과값이 있을텐데 그 둘의 변화량인 차이에 집중하자는 얘기이다. 최종 결과를 산출하는 근본적인 집합(underlying mapping)을 $H(x)$라고 보고 입력을 $x$라고 볼 때 residual mapping $F(x)$는 $H(x)-x$로 표현될 수 있다. 그 다음으로 original(underlying) mapping인 $H(x)$는 다음 아래와 같이 나타낼 수 있다.

$$
F(x)+x
$$

ResNet논문에서 degradation problem이 일어나는 상황은 $H(x)$와 $x$의 차이가 거의 나지 않을때라고 말한다. 즉, $F(x)$가 0에 근접하며 **residual mapping 레이어들이 미세한 조정값을 만들어 낼 때**를 다루고 있다. 다시 말해 입력(identity)하고 미세하게 조정된 값(residual)을 따로 분리해서 보고 둘을 합쳐 underlying mapping을 만들어 내는 것이 resnet이 제안하는 방안이다. 이때 이 $x$를 더하는 것을 identity mapping(shortcut connection)이라고 표현하며 이런 기법을 **잔차 연결(residual connection), 혹은 skip connection** 이라고 말을 한다. 표현하는데 특별한 파라미터를 요구하지도 않으며 계산이 복잡하지도 않는다. 더 나아가 역전파 시 gradient가 shortcut connection을 타고 직접 흐르기 때문에 깊은 레이어까지 안정적으로 전달된다. resnet이 전설적인 논문인 점이 여기서 나온다.

### Identity mapping shortcut

$H(x)$를 $y$라고 하면 다음과 같이 표현할 수 있다.

$$
y = F(x, \{W_i\}) + x
$$

이때 x와 y의 차원(dimension)이 맞지 않아서 계산하는데 문제가 발생할 수 있다. 이런 상황에서는 **x**를 $W_s$로 linear projection을 시켜서 y와 차원을 맞추는 작업을 한다.

$$
y = F(x, \{W_i\}) + W_s x
$$

이때 $W_s$는 결국에 변환을 시키는 것이므로 차원이 맞지 않을때만 사용하도록 제한한다. 그리고 $W_s$는 Linear layer로도 되지만 보통은 **1x1 커널사이즈로 convolution layer**을 사용한다.

단, F가 레이어 하나일 경우 단순 linear transformation과 다를 바 없어 residual learning의 이점이 없다. ($y= W_i x + x$)

## Implementation

### VGGNet

vggnet과 resnet은 구현방식이 비슷하지만 둘 다 한 가지가 아닌 아래 그림처럼 여러가지 형태를 만들어야 된다. 하나하나 일일이 하는 것은 매우 번거로운 일이다. [Aladdin Persson의 영상](https://youtu.be/ACmuBbuXn20?si=lew-Wbj-Gp7QUj2z)을 참고했다.

![vggnet2](assets/img/resnet3.png)

위 그림은 VGGNet에 대해 여러가지 모델들을 나타내는 표이다. 먼저 한꺼번에 다 하면 머리가 아프니깐 D(VGG16)을 기준으로 먼저 VGGNet을 표현하자. VGG16을 숫자를 채널사이즈, 'M'을 Maxpool층이라고 했을 때 VGG16은 다음과 같은 어레이로 레이어를 순서대로 표현할 수 있다.

``` python
VGG16=[64,64,'M',128,128,'M',256,256,256,'M',512,512,512,'M',512,512,512,'M']
```

그리고 VGGNet클래스에서 *create_conv_layers*함수를 만들고 위 어레이를 함수의 parameter로 넣는다. 지금까지 VGGNet클래스는 아래와 같이 표현할 수 있다. *in_channels=3*은 그림이 RGB인 3개의 채널임을 의미하고 *num_classes*는 dataset의 class개수이다.

```python

```

이제 conv레이어를 만들어보자. Table1에서 보다시피 A-LRN,C를 제외하면 다 **Conv3-n**을 다루고 있는것을 볼 수 있다.

conv레이어에선 in_channel, out_channel은 서로 꼬리물기하면서 간다. 이건 텍스트로 설명하지 않아도 아래 코드만 봐도 이해가 된다. 그리고 논문과 달리 **Batch Normalization**층을 추가한 것을 볼 수 있는데, 이는 ResNet논문 초반에서 나온 **기존 degradation problem 해결 기법**을 적용한 것이라 볼 수 있다. else에는 그냥 maxpooling계층을 추가하면 된다.

```python
def create_conv_layers(self,architecture):
    layers=[]

    in_channels=self.in_channels
    for x in architecture:
      if type(x)==int:
        out_channels=x

        layers+=[
            nn.Conv2d(in_channels=in_channels,out_channels=out_channels,kernel_size=(3,3),stride=(1,1),padding=(1,1)),
            nn.BatchNorm2d(x),
            nn.ReLU()
        ]

        in_channels=x
        
      else:
        layers+=[nn.MaxPool2d(kernel_size=(2,2),stride=(2,2))]

    return nn.Sequential(*layers)
```

이제 VGG16 말고도 다른 type의 VGG도 받게 하자. 먼저 VGG_types라는 딕셔너리를 만들어 여러 VGG모델을 저장시키고 VGGNet클래스를 아래 코드와 같이 변형시키면 VGGNet 구현은 끝난다.

```python
VGG_types={
    'VGG11':[64,'M',128,'M',256,256,'M',512,512,'M',512,512,'M'],
    'VGG13':[64,64,'M',128,128,'M',256,256,'M',512,512,'M',512,512,'M'],
    'VGG16':[64,64,'M',128,128,'M',256,256,256,'M',512,512,512,'M',512,512,512,'M'],
    'VGG19':[64,64,'M',128,128,'M',256,256,256,256,'M'`,512,512,512,512,'M',512,512,512,512,'M']

class VGG_net(nn.Module):

    self.fully_connected=nn.Sequential(
        nn.Linear(512*7*7,4096),
        nn.ReLU(),
        nn.Dropout(p=0.5),
        nn.Linear(4096,4096),
        nn.ReLU(),
        nn.Dropout(p=0.5),
        nn.Linear(4096,num_classes)
    )

  def forward(self,x):
    x=self.conv_layers(x)
    x=x.reshape(x.shape[0],-1)
    x=self.fully_connected(x)
    return x
    

  def create_conv_layers(self,architecture):
    layers=[]

    in_channels=self.in_channels
    for x in architecture:
      if type(x)==int:

            nn.Conv2d(in_channels=in_channels,out_channels=out_channels,kernel_size=(3,3),stride=(1,1),padding=(1,1)),
            nn.BatchNorm2d(x),
            nn.ReLU()

```

model을 평가(evaluation)할 때는 마지막에 fully_connected대신에 커널 사이즈가 1x1인 convolutional layer을 사용한다고 논문에 나와있다. 근데 이건 확률 0.1%라도 올릴려는 버저비터 성격이 띄므로 이거까진 구현하지 않았다.

### ResNet

다음은 resnet인데 구조는 아래 테이블 같이 표현할 수 있다.

![resnet4](assets/img/resnet4.png)

18,34레이어에 모델에서는 residual mapping이 convolutional layer 2개로 이루어져있는 것을 볼 수 있지만 나머지 모델에서는 conv layer가 3개씩 있는 것을 볼 수 있다. 그래서 이 모델을 구현할 때 논문에서도 처음에 **Basic block**과 **Bottle Neck**블럭으로 각각 나눠서 처리하는 것을 볼 수 있다.

identity_downsample을 identity mapping을 처리해주는 기능을 한다. stride=1 (차원이 변하지 않음) 이면 nn.Sequential()로 입력 그대로 보내주는 기능을 하고 stride$\neq$1 (차원이 변함)이면 그에 맞는 stride를 적용해 입력값의 차원을 조정해준다. 예를들어 위 논문 및 표에 나온대로 적용하면 stride=2이다. 가로 세로가 절반씩 되어야 하기 때문이다.

#### Basic Block

![resnet5](assets/img/resnet5.png)

*expansion*이라는 변수를 할당할 건데, 위 표를 보면 basic block은 64로 끝나면 bottle neck은 그것의 4배인 256으로 끝난다. 이런 점을 처리하기 위해서 *expansion*이라는 변수를 할당한다. 더 깊은 이해를 위해 residual mapping 부분을 보자.

```python
expansion=1
self.residual=nn.Sequential(
  nn.Conv2d(in_channels=in_channels,out_channels=out_channels,kernel_size=3,stride=stride,padding=1),
  nn.BatchNorm2d(out_channels),
  nn.ReLU(),
  nn.Conv2d(in_channels=out_channels,out_channels=out_channels*BasicBlock.expansion,kernel_size=3,stride=1,padding=1),
  nn.BatchNorm2d(out_channels*BasicBlock.expansion)
)
```

basic block은 채널수가 다 같으므로 expansion=1이고 이미지 크기 조정하는 stride는 앞에 있는 convolutional에 이루어 지는것을 볼 수 있다.

*identity_downsample*은 위에 있는 것을 이 클래스에 그대로 끌고 온 것이라 보면 된다. 코드로 표현하면 아래와 같다.

```python
class BasicBlock(nn.Module):
  expansion=1
  def __init__(self,in_channels,out_channels,identity_downsample,stride=1):
    super(BasicBlock,self).__init__()
    self.identity_downsample=identity_downsample
    self.residual=nn.Sequential(
        nn.Conv2d(in_channels=in_channels,out_channels=out_channels,kernel_size=3,stride=stride,padding=1),
        nn.BatchNorm2d(out_channels),
        nn.ReLU(),
        nn.Conv2d(in_channels=out_channels,out_channels=out_channels*BasicBlock.expansion,kernel_size=3,stride=1,padding=1),
        nn.BatchNorm2d(out_channels*BasicBlock.expansion)
    )

    self.relu=nn.ReLU()
    
  def forward(self,x):
    identity=x
    x=self.residual(x)
    identity=self.identity_downsample(identity)
    x+=identity
    x=self.relu(x)

    return x
```

#### Bottle Neck

![resnet6](assets/img/resnet6.png)

Bottle neck은 입력의 채널보다 더 낮은 곳으로 보내서 convolutional하는 것을 볼 수 있다. 더 작은 input/output dimension에서 처리하여 시간복잡도를 줄이는 방식으로 설계된 것이다. stride도 2번째 convolution에서 처리한다. botlle neck은 다음과 같이 나타낸다.

```python
class BottleNeck(nn.Module):
  expansion=4
  def __init__(self,in_channels,out_channels,identity_downsample,stride=1):
    super(BottleNeck,self).__init__()
    self.identity_downsample=identity_downsample
    self.residual=nn.Sequential(
        nn.Conv2d(in_channels=in_channels,out_channels=out_channels,kernel_size=1,stride=1,padding=0),
        nn.BatchNorm2d(out_channels),
        nn.ReLU(),
        nn.Conv2d(in_channels=out_channels,out_channels=out_channels,kernel_size=3,stride=stride,padding=1),
        nn.BatchNorm2d(out_channels),
        nn.ReLU(),
        nn.Conv2d(in_channels=out_channels,out_channels=out_channels*BottleNeck.expansion,kernel_size=1,stride=1,padding=0),
        nn.BatchNorm2d(out_channels*BottleNeck.expansion)
    )

    self.relu=nn.ReLU()

  def forward(self,x):
    identity=x
    x=self.residual(x)
    identity=self.identity_downsample(identity)
    x+=identity
    x=self.relu(x)

    return x
```

여기서 *expansion*이 out_channel의 4배 이므로 4인것을 볼 수 있다. 눈치가 빠르다면 Bottle neck은 basic block과 달리 입력의 차원에 대한 처리를 block 들어가자마자 바로 해야된다는 것을 알 수 있다.

#### Overall

각각의 블록을 봤으니 *_make_layer* 함수를 완성시키자. *block*인수는 basic block, bottle neck block 같은 class 이므로 각각의 expansion을 끄집어 낼 수 있다. 그리고 in_channel의 결과를 out_channel*expansion으로 처리하여 조정한다. 코드는 아래와 같이 추가할 수 있다. 직접 설명보다 코드를 보며 이해하는것이 더 이해하기 쉬울것이다.

```python
def _make_layer(self,block,num_residual_blocks,out_channels,stride):
    strides = [stride] + [1] * (num_residual_blocks - 1)
    layers=[]

    for stride in strides:

      identity_downsample=nn.Sequential()
      expansion=block.expansion

      if stride!=1 or self.in_channels!=out_channels*expansion:
        identity_downsample=nn.Sequential(
            nn.Conv2d(in_channels=self.in_channels,out_channels=out_channels*expansion,kernel_size=1,stride=stride),
            nn.BatchNorm2d(out_channels*expansion)
        )
        
      layers.append(block(self.in_channels,out_channels,identity_downsample,stride))
      self.in_channels=out_channels*expansion

    return nn.Sequential(*layers)
```

layers인자는 conv레이어마다 block의 수를 저장해 놓은 list이다. 예를들어 50-layer는 [3,4,6,3]으로 저장되어 있다. 간단한 이해를 위해 아래 코드만 긁었다.

```python
def __init__(self,block,layers,in_channels=3,num_classes=10):

  # ...
  
  self.conv2=self._make_layer(block,layers[0],out_channels=64,stride=1)
  self.conv3=self._make_layer(block,layers[1],out_channels=128,stride=2)
  self.conv4=self._make_layer(block,layers[2],out_channels=256,stride=2)
  self.conv5=self._make_layer(block,layers[3],out_channels=512,stride=2)

  # ...

```

전체 Resnet코드는 다음과 같다.

```python
class ResNet(nn.Module):
  def __init__(self,block,layers,in_channels=3,num_classes=10):
    super(ResNet,self).__init__()
    self.in_channels=in_channels
    self.conv1=nn.Sequential(
        nn.Conv2d(in_channels=in_channels,out_channels=64,kernel_size=7,stride=2,padding=3),
        nn.BatchNorm2d(64),
        nn.ReLU(),
        nn.MaxPool2d(kernel_size=3,stride=2,padding=1)
    )
    
    self.conv2=self._make_layer(block,layers[0],out_channels=64,stride=1)
    self.conv3=self._make_layer(block,layers[1],out_channels=128,stride=2)
    self.conv4=self._make_layer(block,layers[2],out_channels=256,stride=2)
    self.conv5=self._make_layer(block,layers[3],out_channels=512,stride=2)

    self.avgpool=nn.AdaptiveAvgPool2d((1,1))
    self.fc=nn.Linear(512*block.expansion,num_classes)

  def forward(self,x):
    x=self.conv1(x)
    x=self.conv2(x)
    x=self.conv3(x)
    x=self.conv4(x)
    x=self.conv5(x)
    x=self.avgpool(x)
    x=x.reshape(x.shape[0],-1)
    x=self.fc(x)

    return x

  def _make_layer(self,block,num_residual_blocks,out_channels,stride):
    strides = [stride] + [1] * (num_residual_blocks - 1)
    layers=[]

    for stride in strides:

      identity_downsample=nn.Sequential()
      expansion=block.expansion

      if stride!=1 or self.in_channels!=out_channels*expansion:
        identity_downsample=nn.Sequential(
            nn.Conv2d(in_channels=self.in_channels,out_channels=out_channels*expansion,kernel_size=1,stride=stride),
            nn.BatchNorm2d(out_channels*expansion)
        )
        
      layers.append(block(self.in_channels,out_channels,identity_downsample,stride))
      self.in_channels=out_channels*expansion

    return nn.Sequential(*layers)
```
